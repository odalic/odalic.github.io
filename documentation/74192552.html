ADEQUATE : Story: User may lock the subject column

Created by Václav Brodec, last modified by Tomas Knap on Sep 23, 2016

Github Issues: https://github.com/odalic/sti/issues/31 , https://github.com/odalic/odalic-ui/issues/69

Req: 74,15

 

User Story

User may lock the subject column as depicted below on the mockup. 

Preconditions

  • The algorithm has been run at least once on the data or the application already knows how many columns are available in the input and which are ignored.

Detailed Description

  1. In the Step "subject column detection", user may lock the subject column.
    1. As a result, next executions of Odalic Core must not change such column. 

 

Alternative

  1. User may click on any other column (to select differnet column)
    1. As a result, such column is selected
    2. At the same time, such column is also locked. 

Acceptance Criteria & Test Scenarios

  1. I can go to Subject column detection step, switch to a different column
    1. as a result, such column is locked. 
  2. As a result, next run of Odalic Core uses such subject column in all KBs
