ADEQUATE : Scenario 2a Knowledge base import

Created by Jan Váňa, last modified on Mai 23, 2016

Initial State

The user is logged on the system  (see: Scenario 0a Login and home screen)

Description

  1. User displays a list of knowledge bases (see Scenario 9)   
  2. User decides to add a new knowledge base
    1. As a result, system shows the dialog from Mockup 1.
  3.  User inserts the knowledge base name and description and confirms the dialog. 
    1. As a result, system creates a new knowledge base.
  4. The scenario continues at step 3 of either Scenario 2b, or Scenario 2c, depending on whether the user wants to create external or internal dataset in the new knowledge base.

Alternatives

  1. User chooses to manage knowledge base in PoolParty in step 3.
    1. As a result, system import datasets from the PoolParty.

What can go wrong

  • There is already a knowledge base with the same name.

System state on completion

The system saves the knowledge base and the user can use it in further conversion tasks.

Requirements Motivating the Scenario

48, 49.

Implementation Notes

RDF files are stored into the internal RDF store.
