ADEQUATE : Story: User can see results of classification/disambiguation

Created by Tomas Knap, last modified by Václav Brodec on Jul 14, 2016

Github Issue: https://github.com/odalic/odalic-ui/issues/20

User Story

As a user i want to be able to see result of running classification/disambiguation

Preconditions

Detailed Description

  1. User can see results of the classification/disambiguation in a table
    1. See the mockup below, which is an excerpt from the big picture mockup in Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification
    2. System shows with differently colored tags classes/entities the given colum/cell was annotated with
      1. Each knowledge base has its color
      2. The tag with the class/entity from the primary knowledge base is always on the first position and emhasized, e.g. with bolded font. 
      3. By default, the system shows one class/entity from each knowledge base (if it happens there is no suitable candidate, there is no tag from the given knowledge base)

 


Acceptance Criteria & Test Scenarios

  1. Specify DBPedia knowledge base
  2. Specify some generic SPARQL repository with couple of classes/entities/rels manually added into
    1. Mark the generic SPARQL repository as being primary one.
  3. Run Odalic
  4. Ensure that tags from both knowledge bases appear correctly
