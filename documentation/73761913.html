ADEQUATE : Story: User can configure source of the task - remote file location

Created by Tomas Knap, last modified on Sep 23, 2016

Github Issue: https://github.com/odalic/odalic-ui/issues/22 / https://github.com/odalic/odalic-ui/issues/19 / https://github.com/odalic/odalic-ui/issues/18

User Story

As a user i want to either directly upload new file, select already uploaded file or define remote location of the file to be processed.

Possibility to point to remote file is missing

 Preconditions

Detailed Description

  1. User clicks "new task"

    1.  for the overall description of the scenario, please see: Scenario 1 Task creation

    2. the part responsible for selecting the source file should look as follows: 


  2. User may click upload and select file to be uploaded
    1. As a result, system shows the name of the uploaded file and pre-fills the identifier automatically with the last part of the file location (file name). User may adjust the identifier if needed.
    2. Note: Identifier is read only (cannot be filled) before the file is uploaded. 

Acceptance Criteria & Test Scenarios

Variations

In Step 2, user may select existing file - in that case "Identifier" section is hidden

In Step 2, user may specify file location. In that case, file is not downloaded now, but only when the task is executed. Identifier is hidden in this case as the file is downloaded as the task is executed and not available otherwise.
