Issues/Bugs

Task execution ended by error. The UI displayed the error statement, but user cannot recognize what is the reason of the error:

In the original file text fields (in the first column) are wrapped in quotes). Currently the program is not prepared for this file configuration. The error was not thrown during input file processing. Then the error was caused by SPARQLProxy and the method queryEntityCandidates when the query string contained quotes, These strings are problematic for this query method. Then I removed all quotes and file was processed.

  • issue: https://github.com/odalic/sti/issues/242 (error when quotes in query)
  • issue: https://github.com/odalic/sti/issues/245 (support CSV format configuration: text fields are wrapped in quotes)

Special German characters, e.g. ä, not correctly displayed

  • issue: https://github.com/odalic/sti/issues/243

UI - result - first step: not all rows are displayed (just 47 from 85 rows are displayed), this leads to not proper highlighting of even and odd rows (due to missing rows)

  • issue: https://github.com/odalic/odalic-ui/issues/183
