User Story

As a user I want to import a configuration for the current task from a file in order to allow me to restore from backup or move the processing from different instance.



Preconditions

  • A valid export file is available to the user.

Detailed Description

  1. User clicks the button for importing a task configuration.
  2. An alert appears warning the user about losing all unsaved data.
  3. User confirms the import.
  4. Open file dialogue appears.
  5. User chooses a location to load the file from.
  6. The file is uploaded to the server.
  7. Current task is set to the values contained in the configuration.

Acceptance Criteria & Test Scenarios

  1. Application will not allow to load a corrupted or syntactically incorrect file and keeps the original configuration intact in that case.
  2. Application successfully handles export/import cycle.
