ADEQUATE : Scenario 0b Signing up

Created by Václav Brodec, last modified on Mai 10, 2016

Initial State

The not yet logged user arrives to the sign up page (either by clicking the sign up button or following any sign up recommendation link). If he or she already pre-filled the login, it is already filled where appropriate on the sign up screen.

Description

  1. The sign up page contains a form depicted in Mockup 1.

  2. User needs to input the login, e-mail and password (twice for confirmation).

  3. After the submit, an email (Mockup 2) is sent to the provided e-mail address with temporary confirmation link.
  4. The user confirms the sign up by following the link. Only then he is able to log in.
  5. From the confirmation page he can continue to login or is redirected to login page in few seconds.

Alternatives

What can go wrong

  • The provided e-mail does not exist.
  • The user does not confirm the registration.
  • The confirmation link has already expired when the user attempts to follow it. 

System state on completion

  • User is able to log in.
