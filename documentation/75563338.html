ADEQUATE : Story: User may remove suggested relation

Created by Tomas Knap, last modified by Václav Brodec on Okt 05, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/110

Req: 77

User Story

In the dialog (detail) of any relation, it must be possible to have NO selected resource for certain, or all KBs (e.g. by clicking on the already selected resource it gets deselected)

Preconditions

  • The algorithm has been run at least once on the data or the application already knows how many rows and columns are available in the input.

Detailed Description

  1. Use may remove any single relation fromt he graph
    1. by opening detailed dialog for any edge and removing relations there  (so it must be allowed to deselect one or all suggested relations).
    2. if all relations are removed, edge must disappear. 

Acceptance Criteria & Test Scenarios

  1. User may go to relation discovery step (Step 3), click on the edge and remove suggested relation from the dialog. 
    1. If the user removes all relations, than the edge must disappear
