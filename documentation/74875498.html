User Story

As a user I am able to select one of the existing available files (both local and remote) and let the application display (or provide by the API) the associated meta-data.

I am able to alter the meta-data.

Preconditions

  • At least one existing file definition is available to the user.

Detailed Description

  1. User lists the existing files and selects one of them.
  2. User changes the meta-data and then confirms the change (by hitting the update button) or, if using the API, uploads a new complete version of meta-data in one step.
  3. The old version of meta-data is immediately replaced by the updated one in the permanent storage. The change is not confirmed until it is done so.
  4. User is informed about the successful update.
  5. When the user selects the file (by its identifier) later, the new meta-data are displayed (or provided in the API call result).
  6. All subsequent usage of the file in referring tasks employs the new meta-data.

Acceptance Criteria & Test Scenarios

  1. Updated meta-data remain available to the user in the same form even after server restarts and/or session renewal (if implemented).
  2. Restored meta-data are identical to the one updated.
  3. Only valid meta-data are updated.
