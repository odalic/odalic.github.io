ADEQUATE : Learning from user feedback

Created by Tomas Knap, last modified on Dez 07, 2016

In general, the algorithm should learn from feedbacks provided by the users. 

Ideas:

  • When certain classification is marked as being wrong (alternative is selected) for file X, column C, then we can learned from that and in the future penalize such classification in the future for the documents with the same/similar structure. 

  • When certain classification is marked as being wrong (alternative is selected) for file X, column C, we can learn from that and in the future prefer such suggested classification for the documents with the same/similar structure. 
  • Use the already executed classifications/disambiguations/relations discoveries as a learning set of cases in supervised learning and try to deduce classifications/disambiguations/relations discoveries for the other cases based on that.
  • ?

Different types of feedbacks

  • User selected different 
    • classification for certain column
    • disambiguation for certain cell value
    • type of relation for certain relation
  • (not supported now) User marked as wrong (but does not provide an alternative) for certain:
    • disambiguation 
    • relation
    • ?classification
