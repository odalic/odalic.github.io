ADEQUATE : Story: Export resulting data as CSV data

Created by Tomas Knap, last modified by Václav Brodec on Jul 19, 2016

Github Issue: https://github.com/odalic/sti/issues/82 / https://github.com/odalic/sti/issues/81

User Story

As a user i want to export data as CSV (with classfied columns, disambiguated entities, and denoted relations), so that I can publish such data later as, e.g., Linked Data.

Preconditions

The user is logged on the system (see: Scenario 0a Login and home screen)

User have defined and run classification/disambiguation/relation discovery task (see: Scenario 1 Task creation). User is observing the results of the classification/disambiguation/relation discovery task. User reached the third step - relations specification. 


Detailed Description

  1. User clicks Export to CSV

  2. As a result, system exports the given data (including the feedback provided) as CSV data (TODO think about the particular columns being exported by default). Together with the data, also CSV on the Web metadata is exported (JSON file) - a sample how the exported data looks like is depicted here. 

  3. User may download: 
    1. the exported CSV data (with extra columns holding disambiguated entities)
    2. the CSV on the Web metadata (json) (holding classifications, relations, ...)

Acceptance Criteria & Test Scenarios

  1. Run Odalic Core task
  2. Ensure that there is subject column and at least one relation to one another column. 
  3. Click Export to CSV data
  4. Check that: 
    1. all files are included and properly formatted
