ADEQUATE : Scenario 6b Task configuration - Columns

Created by Václav Brodec, last modified on Jun 19, 2016

Initial State

The logged-in user is either creating a task (from scratch or based on some pre-existing) or reviewing configuration of some scheduled task.

Description

  1. When the user selects Columns tab, appropriate group appears (Mockup 1).

  2. User can provide custom name of the column and own suggestion for concepts (using the same means as in Scenario 4b).

Alternatives

  • User can decide to not disambiguate all cells in the column (they will be treated as literals).
  • User can decide to not classify the column. This has the same effect as if the column has never existed.
  • When the dialog is displayed during task creation, only Apply button is available. 
  • When the dialog is displayed for already running task, only Apply in new task button is available. 
  • If applied to already running task (which entered this state after dialog appearance), the dialog behaves as if Apply in new task button was clicked (notifying the user about it in the process).

What can go wrong

  • The concept visualization relying on remote access can suffer from loss of connection or service outage.

System state on completion

  • Classification hints (and disambiguation hints for all cells in the not disambiguated columns) are set for the currently processed configuration.
