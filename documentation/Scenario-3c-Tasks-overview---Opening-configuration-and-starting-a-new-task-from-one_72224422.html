ADEQUATE : Scenario 3c Tasks overview - Opening configuration and starting a new task from one

Created by Kateřina Boková, last modified by Václav Brodec on Mai 26, 2016

Initial State

The user is logged on the system (see: Scenario 0 Login and home screen).

The user has executed a task (see: Scenario 1 Task creation).

Description

  1. User can choose to see the defined tasks.
  2. Then the user sees a table of his own (or in the case of administrator all present) tasks and a filter inputs. Information about specific task is located in a row of table. A row of table has following format:
    user's id, id/name of task, state of task; reference to configuration; result of the task; task commands. Possible values for the columns are following: state of task (cancelled, running , finished, waiting), task commands (run, cancel, remove),

    1. As a result, system is in situation depicted in the mockup My tasks .

  3. The user can click on item Configuration to display the task configuration.
    1. As a result, a dialog depicted in Scenarios 6 a, b, c, d, e, f appears.
  4. The user can start new task with pre-filled configuration adapted from the chosen task. 
    1. See Scenario 1 Task creation and Scenarios 6 a, b, c, d, e, f.

Alternatives

What can go wrong

  • Database with history of tasks is not available.

System state on completion

  • The system is in the same state.
