ADEQUATE : Number of new babies by the cities, districts

Created by Tomas Knap, last modified on Dez 14, 2016

File: http://service.stmk.gv.at/ogd/OGD_Data_ABT17/statistik/STMK_2012_LIVEBIRTHS.csv (reduced to 20 lines)

Analysis of the results

against Jan's (reduced) DBpedia, original DBpedia and German DBpedia:

Classification:

  • NUTS1 not; NUTS2 only DBpedia, but strange (Item); NUTS3 only German DBpedia, but without label
  • DISTRICT_NAME all, but only DBpedia seems correct (District)
  • LAU_NAME all, but strange, just reduced DBpedia seems better (Settlement)
  • DISTRICT_CODE, LAU_CODE, LIVEBIRTHS and YEAR not (expected, just numeric values)

Disambiguation:

  • NUTS1 all seems OK; NUTS2 only DBpedia, but seems strange; NUTS3 nothing
  • DISTRICT_NAME mostly all seems OK, just Graz-Stadt seems strange
  • LAU_NAME sometimes all OK, sometimes just German DBpedia, sometimes nothing
  • DISTRICT_CODE, LAU_CODE, LIVEBIRTHS and YEAR are just numeric values

Subject column:

  • LAU_NAME always - seems OK

Relations:

  • LAU_NAME -> DISTRICT_NAME (district) seems OK (except German DBpedia)
  • LAU_NAME -> NUTS3 (nutsCode) only in German DBpedia, but not correct

Complete data (original and reduced file, screens, exports and caches):

  • STMK_2012_LIVEBIRTHS.zip

Further notes from Tomas Knap (tested 1.12.)

- It correctly disambiguates NUTS1 column
- identifies relation between LAU_NAME and DISTRICT_NAME - http://dbpedia.org/ontology/district

- There are certain resources which are not available in dbpedia but available in german dbpedia. Which allow more disambiguated entities, e.g. in case of http://de.dbpedia.org/page/Lannach

 

Issues:
- DISTRICT_NAME is classified as "http://dbpedia.org/resource/article". This is extremaly strange as the disambiguations for http://dbpedia.org/resource/Deutschlandsberg contain quite some useful classes
- Disambiguation - relying for one resource during disambiguation is certainly not correct! For example, Hollenegg is disambiguated to Schloss_Hollenegg. But there is better match: http://dbpedia.org/page/Hollenegg!
- Categorie http://de.dbpedia.org/resource/Kategorie:Gundersdorf is used to disambiguate city of Gundersdorf, which is not correct. Again, more candidates would be helpful. Categories should not be used!
- NUTS1 is disambiguated, but there is no class.
- NUTS1 is disambiguated, but NUTS2, NUTS3 NOT!
- NUTS3 is classified with http://dbpedia.org/ontology/nutsCode, which is owl:DatatypeProperty
- There is no label for http://dbpedia.org/ontology/nutsCode, but such resource contains rdfs:label (e.g.). For http://dbpedia.org/ontology/Settlement, the label is fetched!
- identifies relation between LAU_NAME and NUTS3, but it is not correct
