ADEQUATE : Story: Specifying primary knowledge base

Created by Tomas Knap, last modified by Václav Brodec on Jul 19, 2016

Github Issue: https://github.com/odalic/sti/issues/91 / https://github.com/odalic/sti/issues/92 / https://github.com/odalic/odalic-ui/issues/27

User Story

As a user i want to be able to specify primary knowledge bases (via config file)

Preconditions

Detailed Description

  1. User specifies one of the configured knowledge bases as primary one in the  config file of the Odalic Server application
    1. e.g. some generic sparql repository with couple of classes/entities/rels manually added into
  2. After the system is restarted, classification, disambiguation, relation discovery is executed against both knowledge bases, but the annotations from the primary kb are preferred

Acceptance Criteria & Test Scenarios

  1. Specify DBPedia knowledge base
  2. Specify some generic SPARQL repository with couple of classes/entities/rels manually added into
    1. Mark the generic SPARQL repository as being primary one.
  3. Run Odalic
  4. Ensure that terms from the generic SPARQL repo are preferred when there are also annotations from DBpedia
