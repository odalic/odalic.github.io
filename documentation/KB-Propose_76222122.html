ADEQUATE : KB Propose

Created by Tomas Knap, last modified on Nov 27, 2016

 

Settings in dbpedia.config:

# Settings for inserts
kb.insert.supported=true
kb.insert.prefix=http://odalic.eu/concept/
kb.insert.root.class=http://www.w3.org/2002/07/owl#Thing
kb.insert.label=http://www.w3.org/2000/01/rdf-schema#label
kb.insert.alternative.label=http://www.w3.org/2000/01/rdf-schema#label
kb.insert.subclass.of=http://www.w3.org/2000/01/rdf-schema#subClassOf
kb.insert.instance.of=http://www.w3.org/1999/02/22-rdf-syntax-ns#type
kb.insert.class.type=http://www.w3.org/2002/07/owl#Class
kb.insert.graph=http://propose.odalic.eu

 

Proposing entity (not a class):

When running Odalic on top of Books-v1.csv, I just click on propose for Steven Erikson. As a result I got: 

http://odalic.eu/concept/steve-eric http://www.w3.org/1999/02/22-rdf-syntax-ns#type http://www.w3.org/2002/07/owl#Thing
http://odalic.eu/concept/steve-eric http://www.w3.org/2000/01/rdf-schema#label "Steve-custom"
http://odalic.eu/concept/steve-eric http://www.w3.org/2000/01/rdf-schema#label "Steve-custom2"

 

Problem to be solved: The first triple should use as the object (the class the new entity is instance of) the real class from the primary knowledge base which is used for classifying the column (and not the pre-configured root class).

If the class is not known (there is no classification from the primary knowledge base), such propose should fail with a message (it is necessary to first select/propose class for the column from the primary knowledge base).

 

Extension needed: owl:equivalentClass (configurable term in config.properties for the primary KB) to the classes from non-primary KBs.
