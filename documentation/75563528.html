ADEQUATE : Story: User may lock relations (at the level of all relations between two nodes)

Created by Tomas Knap, last modified by Václav Brodec on Okt 05, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/110 / https://github.com/odalic/sti/issues/149

Req: 74

User Story

As a user I want to be able to lock the selected relations for the given two nodes. I can do that either from canvas or from the dialog

The choice is remembered for the re-runs.


Preconditions

  • The algorithm has been run at least once on the data or the application already knows which cells are available in the input.

Detailed Description

  1. User want to lock the given relations so that next runs of the algorithm do now overwrite such selection.
  2. In Step "relation discovery", user clicks on the relation and then clicks on the lock icon in top right corner of the dialog (alternatively, user may also click on the icon on the edge)
    1. As a result, icon changes from unlocked to locked
    2. The choice is saved for the re-runs
  3. The user re-runs the algorithm.
  4. The icon for relation must correspond to the actual state as it was before re-run (locked/unlocked)
  5. User may unlock the given relation 
    1. As a result, icon changes from locked to unlocked
    2. The user is able to take his decision back without any problems affecting the immediate export. No re-run is necessary - So by locking/unlocking, no auomtatic re-run is needed. 

Acceptance Criteria & Test Scenarios

  1. User may lock certain relation, icon is changed
  2. User may re-run alogirhtm 
  3. As a resul, icon remains correclty set up and odalic core does not try to re-evaluate any relation between these two columns
  4. User may un-lock,icon is properly changes
