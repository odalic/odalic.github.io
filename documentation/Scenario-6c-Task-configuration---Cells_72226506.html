ADEQUATE : Scenario 6c Task configuration - Cells

Created by Václav Brodec, last modified on Mai 22, 2016

Initial State

The logged-in user is either creating a task (from scratch or based on some pre-existing) or reviewing configuration of some scheduled task.

Description

  1. When the user selects Cells tab, appropriate group appears (Mockup 1).

  2. User can add another item to customize a cell disambiguation or prevent any attempts to do so (item identified by both the column order (or name) and a literal).

  3. User can provide the custom concept using the same means as in Scenario 4c.
  4. When user unchecks the box in Disambiguate column, the concept is deleted and the hints stand for "do not attempt to disambiguate".

Alternatives

  • User can delete the disambiguation hint by clicking on "delete" in the last column. 
  • When the dialog is displayed during task creation, only Apply button is available.  
  • When the dialog is displayed for already running task, only Apply in new task button is available. 
  • If applied to already running task (which entered this state after dialog appearance), the dialog behaves as if Apply in new task button was clicked (notifying the user about it in the process).

What can go wrong

  • The concept visualization relying on remote access can suffer from loss of connection or service outage.

System state on completion

  • Cell disambiguation hints are set for the currently processed configuration.

Implementation notes

  • When the input file (or its schema) is loaded, the application uses its values to provide hints for user input.
