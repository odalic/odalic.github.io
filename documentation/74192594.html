ADEQUATE : Story: User ignores a column in a base

Created by Václav Brodec, last modified by Tomas Knap on Sep 30, 2016

Github Issues: https://github.com/odalic/sti/issues/145 , https://github.com/odalic/sti/issues/144

Req: 73, 15

User Story

As a user I want the algorithm to completely ignore some column as if it never existed when processed by a base.

The choice is remembered for the re-runs, so unless I revert the choice back, the column will be ignored.

It is not possible to ignore the chosen subject column for primary base and an ignored column for a primary base cannot be selected as the subject column.

No processing is done on the ignored column by the algorithm.

All controls for the column in the UI are disabled (but their content and other states are kept until the next run) and associated feedback is ignored as long as the column itself is ignored.

Such column is not available in the export.

Preconditions

  • The data or the application identified the available columns or at least their number in the input.

Detailed Description

  1. User indicates that a column is to be ignored.
  2. Associated controls are disabled (but their states and content kept until re-run).
  3. Associated relations are shown to be removed when the relations designing screen is displayed.
  4. All ignored columns are made unavailable for the subject column selection.
  5. The choice is saved for the re-runs.
  6. The associated check-box is filled according to the choice.

Acceptance Criteria & Test Scenarios

  1. The choice to ignore the column is applied in the subsequent export, so such column is not available in the export
  2. It is tested that subject column cannot be marked as ignored
  3. It is tested that in Step 1, ignored column (marked after the previous run of the algorithm) cannot be marked as subject column 
  4. Note the differences between this functionality an the option to leave a column from export, which takes the column into account during the whole run of the algorithm, but leaves it out in the output and all relations it takes part in.
  5. It behaves correctly in case of the sequense - mark column C as ignored, re-run, mark as NOT ignored.
