ADEQUATE : Scenario 7a Input file removal

Created by Václav Brodec, last modified on Mai 21, 2016

Initial State

The user is logged to the application.

Description

  1. User enters the file management screen (Mockup 1).

  2. User clicks the button on the row with chosen file.

    1. As a result, the file is removed and its row disappears.

Alternatives

In 2., user can at first select multiple files and then click the button under the table. All files are then removed and their rows disappear and space is freed.

What can go wrong

  • The user already removed the file in other window, instance or using the API.
  • The application is unable to reach the store.

System state on completion

  • The allocated space of the file and its schema is freed.
  • References from the configurations that used the input as their last are forgotten.

Requirements Motivating the Scenario

  •  12, 13, 14, 15
