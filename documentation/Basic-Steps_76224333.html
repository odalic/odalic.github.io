ADEQUATE : Basic Steps

Created by Tomas Knap, last modified on Dez 15, 2016

See sti-main, TMPOdalicInterpreter

  • Detecting subject column
  • Learning
    • For named entity columns (which are not ignored):
      • Learning.learn()
        • LEARNINGPreliminaryColumnClassifier.runPreliminaryColumnClassifier()
          • get the sample of cells
          • find candidate entities for that sample cells (they will then vote for the initial class)
            • KBProxy.findEntityCandidates(sample.getText());  (SPARQLProxy e.g.)
          • TCellDisambiguator.coldstartDisambiguate()
            • KBProxy.findAttributesForEntity (basically all triples about that entity)
            • EntityScorer.computeElementScores() (using TMPEntityScorer)
            • EntityScorer.computeFinal() (using TMPEntityScorer)
          • TCellDisambiguator.addCellAnnotation()
          • TColumnClassifier.generateCandidateClazz()
          • generatePreliminaryColumnClass()
        • LEARNINGPreliminaryDisamb.runPreliminaryDisamb()
          • for those already disambiguate, run LEARNINGPreliminaryDisamb.reselect() - some of them may not vote for the selected class, so in that case, new selection should be made or there is no disambiguation
          • for the rest, run LEARNINGPreliminaryDisamb.constraintDisambiguation (constrained with the winning class)

            • KBProxy.findEntityCandidatesOfTypes(cell value, winningColumnClazz);  (SPARQLProxy e.g.)

            • also tries to run KBProxy.findEntityCandidates(sample.getText());  (SPARQLProxy e.g.) - so without type restriction, when there are not results on the call above. 
              • This is useful in cases when e.g. the sample was not properly selected and other cell values would like to introduce different type value
          • TCellDisambiguator.addCellAnnotation()
          • TColumnClassifier.updateColumnClazz()
      • Update.update()
        • In iterations (until stabilizes) run:
          • Update.collectAllEntityCandidateIds()
          • TColumnClassifier.updateClazzScoresByDC()
            • ClazzScorer.computeDC()
          • Update.reviseColumnAndCellAnnotations()
            • Update.disambiguate()

              • KBProxy.findEntityCandidatesOfTypes(cell value, winningColumnClazz);  (SPARQLProxy e.g.)

              • also tries to run KBProxy.findEntityCandidates(sample.getText());  (SPARQLProxy e.g.) - so without type restriction, when there are not results on the call above. 
          • Update.checkStablization
      • Relationenumeration.enumerate()
