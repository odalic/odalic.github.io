ADEQUATE : Story: User can get further information about the resource (concept)

Created by Tomas Knap, last modified on Okt 12, 2016

Github Issue: https://github.com/odalic/odalic-ui/issues/108 (more might follow when finished)

Req: 52

User may click on the resource (in the adjusted dialog suggestions for annotations from KBs, currently prepared by Kata, there is no action associated with clicking on the resource) and as a result, HTTP GET is executed for that URL and the result is depicted in the iframe. E.g. as depicted below for http://dbpedia.org/resource/Prague:


So far we can expect that the result is always HTML page (which is true in case of dbpedia resources, will be true also in case of PoolParty KB)

Detailed Description (Browsing for the class)

  1. In Step 2 (classes/disambiguation), user may click on detail. From the detail, user may click on the resource. 
    1. as a result HTTP GET is initiated and the result (HTML description of that resource) is depicted in iframe

 

Alternative

  1. User may click on the resource directly on the Table (not in the dialog with annotation details)
    1. Note: This would require the resource to be clickable also on the Table. 

 

Acceptance Criteria & Test Scenarios

  1. Run Odalic Core
  2. Customize classification, you may click on the resource
    1. as a result HTTP GET is initiated and you may see HTML description of that resource
  3. Browse KB, select alternative class, try to add it. 
  4. Ensure that new class can be selected and after closing customize dialog also appear in the table.
