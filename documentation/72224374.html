ADEQUATE : Story: Handling data cube data

Created by Josef Janoušek, last modified on Feb 01, 2017

Github Issues: https://github.com/odalic/sti/issues/137 , https://github.com/odalic/sti/issues/138 , https://github.com/odalic/odalic-ui/issues/13

Req: 28

See also: Data cube

User Story

As a user i want to define data cube data and export them, so that I can publish statistical data as RDF data cubes.


Preconditions

The user is logged on the system (see: Scenario 0a Login and home screen)

User have defined and run classification/disambiguation task (see: Scenario 1 Task creation). User is observing the results of the classification/disambiguation/relation discovery task.

Detailed Description

  1. User defines new task

    1. as part of that, user defines that the data he would like to process are statistical data. (extra checkbox available) 

  2. User executes the algorithm
    1. Step relation discovery is a bit different (see mockup)
  3. Step 1 (results of classification/disambiguation) is the same as in case of processing non-statistical data
  4. Step 2 is skipped
  5. Step 3 - relation discovery is a bit different (see mockup)
  6. User can export such RDF data (see also Story: Export resulting data as RDF data, but in this case, the data is published according to RDF data cube vocabulary)
    1. System exports data according to RDF data cube vocabulary
    2. Only those dimensions and metrics, which are linked from "observation" node
    3. It uses some default values for data cube name etc.
    4. It uses datatype for measures from ranges of predicates

Changes needed

  • UI
    • add "Statistical data" checkbox to the "Create new task" (task configuration) page (bind to "statistical" boolean field of Configuration) (done)
    • hide the "Step 2" (subject column) of the "Result" page for statistical tasks (when the "statistical" boolean field of Configuration is true)
    • hide the relations graph in the "Step 3" of the "Result" page for statistical tasks (when the "statistical" boolean field of Configuration is true)
    • add new tables for dimensions and measures to the "Step 3" of the "Result" page (according to mockup)
      • when the user changes the component type or predicate entity, send this info as part of feedback
      • predicates will be proposed / searched by the user exactly the same way as normal relations
      • show and set component types and predicates of columns only for primary knowledge base
  • REST API
    • add "IsStatistical" boolean flag to the Configuration of the task (done)
    • add enum to determine whether the column is dimension or measure or nothing of this (done)
    • add statistical annotations to the Result and Feedback in odalic and to Constraints and TAnnotation in sti-main (done)
  • Server
    • algorithm: no relation discovery for statistical data, suggest dimensions and measures or get it from feedback (done)
    • adjust export functions to generate RDF data cube (done)
