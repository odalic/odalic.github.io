User Story

As a user when I provide a feedback (as detailed in the other stories) it must remain to be permanently bound to the associated existing task configuration until its is replaced with its updated version.

When I run a new task from a loaded configuration the bound feedback must be applied to that run.

Preconditions

  • A task has been run once (presumably without any feedback) and the user is exploring its result.

Detailed Description

  1. User either provides a feedback in the UI in parts.
  2. User agrees to re-run in order to send and apply the feedback (a separate controls to only save the feedback without re-running the task are not required)
  3. Application alters the existing feedback (which is empty during the initial task run) according the the provided updated version.
  4. Application re-runs the task applying the feedback.

Acceptance Criteria & Test Scenarios

  1. The altered feedback is present and associated to the task configuration even after server restarts and/or session renewal (if implemented).
  2. Restored feedback is identical to the form it took after the latest modification.
  3. Unless other task options change or the content of input files changes or the used knowledge bases are modified, the loaded feedback gives results differing from the empty or the previous one only to the extent of its additional or altered hints.
  4. Only a valid feedback (adhering to specification) is stored.

Variations

  • User can set the feedback all at once using the API without re-running the task. All criteria apply up to those concerned with the immediate re-run.
