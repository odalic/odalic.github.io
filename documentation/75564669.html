ADEQUATE : Story: Subject column selection for each base

Created by Václav Brodec, last modified on Okt 05, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/104

Req: 

User Story

As a user I would like to see the results of subject column selection for each base and select them manually also per base, as I prefer to let the algorithm choose the best one possible, rather than force the algorithm to select one column across all bases.

So it is necessary to duplicate the current and proposed subject column controls for each base, the primary one should be always displayed as first, but with no special highlights.

Detailed Description

  1. User steps into chosen subject column selection (display) screen.
  2. User checks each automatically selected (or manually set) subject column and either:
    1. does nothing,
    2. switches the selection to manual by either clicking the A/M toggle previously set to automatic mode or selecting other than the previously automatically selected column,
    3. switches the selection to automatic by clicking the A/M toggle previously set to manual mode.
  3. When all is done, proceeds to the available actions (next, previous step; re-run, start from beginning).


Acceptance Criteria & Test Scenarios

  • The screen shows correctly automatically selected columns in the first run.
  • The user is able to select different available columns for each base.
  • A/M switches are respected in repeated runs.
  • Switches react to selecting other than the current column by turning to M state.
