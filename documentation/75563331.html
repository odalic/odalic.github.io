ADEQUATE : Story: User may remove classification/disambiguation from the given cell

Created by Tomas Knap, last modified by Václav Brodec on Okt 05, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/71 / https://github.com/odalic/sti/issues/147

Req: 76

User Story

See Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification - the icon of trash bin in the main table. In the dialog of classification/annoation, it must be possible to have NO selected resource (e.g. by clicking on the already selected resource it gets deselected)

Preconditions

  • The algorithm has been run at least once on the data or the application already knows how many rows and columns are available in the input.

Detailed Description

  1. Use may remove any single classification/disambiguation from the main Table
    1. User may also open detailed dialog for any cell and remove the classification there (so it must be allowed to deselect all classification, disambiguation for the given cell if the user requires so)
    2. Note: Removing the classification/disambiguation will automatically lock the cell, so that any further run of Odalic Core does not modify that cell. 
    3. Note: User may remove all annotations from a single cell if needed. 

Acceptance Criteria & Test Scenarios

  1. User may click on remove button to remove annotation from the table
  2. User may also open dialog for any cell in Table in Step 2 to remove annotation for any of the knowledge bases
