ADEQUATE : Story: User can browse KB for a concept to be used for classification/disambiguation

Created by Tomas Knap, last modified by Václav Brodec on Jul 19, 2016

Github Issue: https://github.com/odalic/sti/issues/63 / https://github.com/odalic/odalic-ui/issues/26 / https://github.com/odalic/odalic-ui/issues/23 / https://github.com/odalic/sti/issues/84

User Story

As a user i want to be able to browse KB to see details/context of the suggested or searched concept.

Preconditions

User can see results of classification/disambiguation (Story: User can see results of classification/disambiguation). 

 

Detailed Description (Browsing for the class)

  1. User can click on "Customize" button next to the column header
    1. To see the location of that button, please see mockup in Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification
    2. As a result, system shows the dialog
      1. The part of the dialog relevant for this story is depicted below
      2. The complete dialog is depicted in Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification
      3. System shows browse button next to each suggested concept. 
  2. User may click on that browse button
    1. As a result, system shows detail of that resource using LodView (see Mockup 2)
      1. User can see the details of the resource, can follow the links to other resources. 
  3. User can switch to graph mode by clicking on the icon in the top right corner 
    1. As a result,system shows graphs with the browsed resource in the center. (Mockup 3)
  4. User may examine relations to other resources using graph mode
  5. User may select resource and request to see the details 
    1. As a result, system is shows the detail mode (Mockup 2) 
  6. User may select any resource by clicking "Add" button in detail overview (Mockup 2)
    1. As a result, the browse dialog is closed and the resource is added to the list of classes suggested. (see Mockup 4)

 


Acceptance Criteria & Test Scenarios

  1. Run Odalic Core
  2. Customize classification
  3. Browse KB, select alternative class, try to add it. 
  4. Ensure that new class can be selected and after closing customize dialog also appear in the table.

Variations

Browsing for the entity disambiguating the cell value

The detailed description works similarly as for browsing for class - the differences:  

  • Different title of the dialog - "Customize disambiguation"

 

Browsing searched resource

Use may also browse (in the same way as above) the resource, which was just searched. See: Story: User can search KB for a concept to be used for classification/disambiguation/relation discovery + the browse button next to the search text field in the Mockup 1

 

Notes: 

Note: As part of Odalic, we provide default browser. Nevertheless, it must be easy to use also custom browsers (e.g. PoolParty (the knowledge base used in AD) already has some sort of wiki browser style, which does not require any login and is read-only and so it must be possible to easily use this browser instead of the default one)

  1. TODO Tomas: Describe in a separate page the use of custom browser and the way how the selected class will be pushed to the list


Implementation Notes: 

LodView has to be extended to have this "Add" button. LodLive has to be extended to properly switch to LodView detail view when certain resource is selected.
