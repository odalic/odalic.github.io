ADEQUATE : Story: User uploads a new file and its meta-data to persistent storage

User Story

As a user I expect the uploaded files and their meta-data (or meta-data describing remote files) to persist between session or server restarts.

Preconditions

  • All the parts of configuration are available and set.

Detailed Description

  1. User commands the server to upload a new file using the UI or the API.
  2. The file is immediately saved to the permanent storage and the upload is not confirmed until it is done so.
  3. User is informed about the successful upload.
  4. The file appears in the list of files available for the user to process.

Acceptance Criteria & Test Scenarios

  1. Uploaded file and/or meta-data remain available to the user even after server restarts and/or session renewal (if implemented).
  2. Restored file and/or its meta-data are identical to the one uploaded.
  3. Only a complete file or valid meta-data are stored.
  4. Both the file and existing meta-data are stored atomically, either both or none.

Variations

  • Similar criteria apply when remote file meta-data are uploaded.
