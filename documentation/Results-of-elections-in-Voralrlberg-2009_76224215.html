ADEQUATE : Results of elections in Voralrlberg 2009

Created by Josef Janoušek, last modified on Dez 13, 2016

Version: develop 11.12.2016
File: http://data.vorarlberg.gv.at/katalog/politik/LT-2009.csv with added semicolon on the first line (reduced to 20 lines)

Analysis of the results

against Jan's (reduced) DBpedia, original DBpedia and German DBpedia):

Classification:

  • Gemeinde and Bezirk all, but differently
  • other columns not (expected, just numeric values) (ÖVP just German DBpedia, but wrongly)

Disambiguation:

  • Gemeinde and Bezirk just very few lines and differently
  • other columns are just numeric values

Subject column:

  • Gemeinde always - seems OK

Relations:

  • Gemeinde -> Bezirk (bezirk / district) seems OK (except German DBpedia)
  • Gemeinde -> ÖVP (areaCode) only in German DBpedia, but not correct

Complete data (original and corrected reduced file, screens, exports and caches):

  • LT-2009.zip

Issues/Bugs

Special German characters, e.g. Ö, not correctly displayed

  • issue: https://github.com/odalic/sti/issues/243

UI - result - first step: not all rows are displayed (just 10 rows are displayed), this leads to not proper highlighting of even and odd rows (due to missing rows)

  • issue: https://github.com/odalic/odalic-ui/issues/183

The original file is not correctly formatted. First line (header) is not ended by semicolon, but other lines are ended by semicolon, so they have one more column (which is empty). So that the task execution ended by error. The UI displayed the error statement, but user cannot recognize, that the error was caused by bad formatting of the input file. Then I added a semicolon to the first line and file was processed.

  • issue: https://github.com/odalic/sti/issues/169 (fixed)

Performance

The file was processed in 7 minutes / was necessary to reduce to 20 lines.
