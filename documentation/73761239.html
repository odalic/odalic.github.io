ADEQUATE : Story: Preview exported RDF data

Created by Tomas Knap, last modified on Jul 06, 2016

Github Issue: 

User Story

As a user i want to preview exported RDF data (one row), so that I can see how the resulting data looks like.

Preconditions

The user is logged on the system (see: Scenario 0a Login and home screen)

User have defined and run classification/disambiguation/relation discovery task (see: Scenario 1 Task creation). User is observing the results of the classification/disambiguation/relation discovery task. User reached the third step - relations specification. 

User clicked Export to RDF (see Scenario 4d). 


Detailed Description

  1. User can see the preview of the data being exported.
    1. As a result, system shows the preview of the data (first row) in the form of a graph (see mockup) 
  2. User may then click Export RDF data (see: Story: Export resulting data as RDF data)


Acceptance Criteria & Test Scenarios

  1. Run Odalic Core task
  2. Ensure that there is subject column and at least one relation to one another column. 
  3. Click Export to RDF data 
  4. Check that preview is depicted

Variations

Instead of clicking "Export RDF data" in Step 2, user may click "Back" - as a result, Step 3 - relation discovery - is depicted.
