skos:broader vs. rdf:type when looking for candidates for classification

When running learning phase - preliminary phase, should we consider as a class of a resource only the attached custom scheme or also skos:broader Concepts?

If considering skos:broader concepts, that can bring more fine grained classification and also that is the way how most thesauri are constracted (just skos:broader, no custom schemas)

On the other hand, custom relations are attached to classes, so if classified as "skos:broader" concept, then, when checking relations, the type of that skos:broader has to be examined !
