ADEQUATE : Story: Use can select alternative/extra classification/disambiguation

Created by Tomas Knap, last modified on Aug 03, 2016

Github Issue: https://github.com/odalic/odalic-ui/issues/32

User Story

As a user i want to be able to select alternative classification/disambiguation

Preconditions

User can see results of classification/disambiguation (Story: User can see results of classification/disambiguation)

Detailed Description (Selecting alternative classification)

  1. User can click on "Customize" button to select alternative classification
    1. To see the location of that button, please see mockup in Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification
    2. As a result, system shows the dialog
      1. The part of the dialog relevant for this story is depicted below
      2. The complete dialog is depicted in Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification
      3. System shows in separate sections Top 10 classifications from each knowledge base (so at maximum 10 classifications with the highest score)
      4. The selected concept for classification is for each knowledge base emphasized 
        1. By default,  when Odalic core runs with no configuration/feedback, it selects for each database the classification with the highest score. If there are more with the same score, it selects an arbitrary one. 
  2. User may select different classification by clicking on different item
  3. User may click close to see the changes
    1. As a result, system must redraw the annotation. 


Acceptance Criteria & Test Scenarios

  1. Run Odalic with at least 2 knowledge bases configured
  2. Try to change classification
  3. Try to change disambiguation
  4. Ensure that changes are properly displayed. 

Variations

Selecting alternative disambiguation

The detailed description works similarly for customizing disambiguation - the differences:  

  • Different title of the dialog - "Customize disambiguation"

 

Selecting extra classification/disambiguation

In Step 3, user may also select multiple classes/entities (so the textareas used must support choice of multiple items)
