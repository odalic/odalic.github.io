ADEQUATE : Scenario 8 Knowledge base - Managing Properties of the Knowledge Base

Created by Tomas Knap, last modified by Václav Brodec on Mai 27, 2016

Initial State

The administrator is logged on the system (see: Scenario 0a Login and home screen)

Description

  1. Administrator would like to change properties of the knowledge base
    1. As a result system shows dialog with list of knowledge bases
  2. Administrator selects the one for which properties should be changed
    1. As a result, system shows properties of that knowledge base
  3. Administrator changes the required properties of the selected knowledge base. For adding new datasets see Scenario 2.
  4. Administrator saves the changes

Alternatives

  1. User is not an administrator.
    1. As a result, the user can edit only his own knowledge bases

What can go wrong

System state on completion

  • Knowlege base properties were changed
