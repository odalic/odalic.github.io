ADEQUATE : Story: Specifying multiple knowledge bases

Created by Tomas Knap, last modified by Václav Brodec on Jul 19, 2016

Github Issue: https://github.com/odalic/odalic-ui/issues/28 / https://github.com/odalic/sti/issues/53

User Story

As a user i want to be able to specify multiple knowledge bases (via config file)

Preconditions

Detailed Description

  1. User specifies 2 knowledge bases in the  config file of the Odalic Server application
    1. DBpedia
    2. Some generic SPARQL repository with couple of classes/entities/rels manually added into
  2. After the system is restarted, classification, disambiguation, relation discovery is executed against both knowledge bases

Acceptance Criteria & Test Scenarios

  1. Specify DBPedia knowledge base
  2. Specify Some generic SPARQL repository with couple of classes/entities/rels manually added into
  3. Run Odalic
  4. Ensure that terms from both knowledge bases are depicted

Variations

Notes

KB to be searched is specified in https://github.com/odalic/sti/blob/develop/configs/Jan/kbsearch.properties : 

kb.search.class=uk.ac.shef.dcs.kbsearch.sparql.DBpediaSearch
