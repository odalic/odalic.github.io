ADEQUATE : Story: User leaves a column from the export

User Story

As a user I want to leave some column and all associated relations from the result, but still keep it as a part of the input when the algorithm is processing it.

The choice is remembered for the re-runs, so unless I revert the choice back, all subsequent exports will miss the column and the relations it forms as well.

I am not able to form additional custom relations with the left out column.

This step is skipped in case of statistical data

Note: When having such extra step, we may need to have "Export" and "Export with adjustment" buttons for RDF/CSV




Preconditions

  • The algorithm has been run at least once on the data or the application already knows how many columns are available in the output.

Detailed Description

  1. User indicates that a column is to be left from the export (by unchecking associated check-box).
  2. Associated relations are shown to be removed when the relations designing screen is displayed.
  3. When the user chooses to export the result, the column (and all relations it appears in) will miss in all available export formats.
  4. The choice is saved for the re-runs.
  5. The associated check-box is filled according to the choice.

Acceptance Criteria & Test Scenarios

  1. All supported export formats do not contain the left out column or relations it forms.
  2. The choice to leave the column is applied also in the subsequent export.
  3. Note the differences between this functionality an the option to entirely ignore a column, which leave the column before the algorithm even starts.

Variations

  • Optional goal: The user is able to take his decision back without any problems affecting the immediate export. No re-run is necessary (unless other feedback, requiring a re-run is provided).
