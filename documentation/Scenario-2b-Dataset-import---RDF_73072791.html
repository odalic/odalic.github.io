ADEQUATE : Scenario 2b Dataset import - RDF

Created by Jan Váňa, last modified on Mai 23, 2016

Initial State

The user is logged on the system  (see: Scenario 0a Login and home screen)

Description

  1. User displays a list of knowledge bases (see Scenario 9)   
  2. User decides to add a new dataset for an existing knowledge base and clicks "Import new data".
    1. As a result, system shows the dialog from Mockup 1.
  3. User inserts the Dataset Name and Named Graph.
  4. User chooses to import from a RDF file and cofirms the dialog.
    1. As a result, system shows the dialog from Mockup 2.
  5. User uploads the RDF file
    1. As a result, the system is in the state depicted in Mockup 3
  6. User confirms the import
    1. As a result, system imports the dataset

What can go wrong

  • System cannot obtain the specified RDF file.

  • The specified file is not a valid RDF file.

  • The knowledge base for the dataset was deleted in the meantime.

System state on completion

The system saves the dataset and the user can use it in further conversion tasks.

Requirements Motivating the Scenario

48.

Implementation Notes

RDF files are stored into the internal RDF store.
