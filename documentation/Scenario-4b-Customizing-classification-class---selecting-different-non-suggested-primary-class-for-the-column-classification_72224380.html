ADEQUATE : Scenario 4b Customizing classification class - selecting different non-suggested primary class for the column classification

Created by Josef Janoušek, last modified by Tomas Knap on Sep 23, 2016

Initial State

The user is logged on the system (see: Scenario 0 Login and home screen)

User have defined and run classification/disambiguation task (see: Scenario 1 Task creation). User is observing the results of the classification/disambiguation task.

Description

  1. User is provided with a table preview with the suggested winning class for the CSV columns

    1. System clearly marks the suggested winning choice (e.g. http://adequate.at/concept/city for column with cities)
  2. User may customize the winning class ("Customize" button below)
    1. As a result, system shows "Customize classification" dialog as below

      1. Dropdown list with alternative classifications suggested by the Odalic Core is shown, for each knowledge base in a different section

  3. User may select a different not suggested concept from the primary knowledge base (ADEQUTe in the mockup below) by clicking on "Browse/Select"
    1. As a result system show dialog, where user may specify the required concept
  4. User selects the concepts required and confirms
    1. As a result system saves such selection and changes the suggested winning concept

Alternatives

  • In 3 user may decide to select one of the suggested concepts (instead of browsing KB and selecting one on its own)
  • In 3. user may create new concept in the given internal knowledge base - as a result, system shows a dialog for specifying basic details about the concept 
  • In 3. user may provide an existing custom concept  - he may write appropriate custom URL to the "custom" text box
  • In 3. when the user isn't satisfied with any suggested alternative, he selects "do not classify" option in the customize dialog, without specifying the correct alternative - as a result, the table preview is updated and there is no classification for the given cell value and in the consequent re-computations, such column is not considered as named entity column being classified/disambiguated.

 

What can go wrong

System state on completion

The user selects the required concept classifying the column.

Requirements Motivating the Scenario

  • 17, 19, 20, 52

Notes

More than one concept (one from each KB) may be used to classify the column

How can the user mark the specified classification/disambiguation as being wrong?  User may simply create/specify a different concept (User has to always specify at least one concept)

It would be nice to have a link next to each candidate concept in the dropdown list, such that when user clicks it, it opens the ontology browser and shows the concept in context (e.g., where it is in the ontology hierarchy etc). 

This would be helpful because sometimes it is not obvious why the candidate concepts get suggested and what they mean.

 

 

From reading related scenarios 4b, 4c, 4d, my understanding is that there is no plan to 'propagate' user feedback, i.e., 'fix' certain annotations on the columns/cells and re-run the algorithm. This is a sensible choice, as propagating feedback can be fairly complex, and could require a fair bit of coding in tableminer+. For the sake of documentation I note it down here as you may consider this for future improvement. Examples:

  • if a winning class is changed, ideally it should trigger a check on the disambiguation results on all cells in that column, and for any cells whose tagged entity does not belong to the new class, they should be revised. Users may be given the option to allow this or not
  • following above, if any cells are revised, it could potentially affect the relations between columns. again users may be given the option to allow this or not
  • The same situation applies if users decide not to annotate the column

Similar 'chain-reactions' can happen if users correct a cell, or relations between cells or columns.

Ultimately, users may be given the option to 'make a few corrections' (column, or cell, or relation) then 're-run the system with my feedback', and go through such steps in a few iterations, like an active learning fashion.
