ADEQUATE : Scenario 6f Task configuration - Import/Export

Created by Václav Brodec, last modified by István Satmári on Mai 22, 2016

Initial State

The logged-in user is either creating a task (from scratch or based on some pre-existing) or reviewing configuration of some scheduled task.

Description

  1. When the user click the "Import configuration..." button (Mockup 1), standard file selection dialogue appears with *.otc (Odalic Task Configuration) suffix filter and any file filter.
  2. User chooses the file with configuration to upload.
  3. Application changes the edited configuration according to the content parsed from the file.

Alternatives

  • User can also export the configuration by clicking the "Export configuration...". The save file dialogue appears (Mockup 2).
  • When the dialog is displayed during task creation, only Apply button is available. 
  • When the dialog is displayed for already running task, only Apply in new task button is available. 
  • If applied to already running task (which entered this state after dialog appearance), the dialog behaves as if Apply in new task button was clicked (notifying the user about it in the process).

What can go wrong

  • The provided file cannot be parsed.

System state on completion

  • The current configuration is set according to the content parsed from the file. 

Implementation details

  • The inner representation of the file is the same as is used in the REST API.
