ADEQUATE : Scenario 0a Login and home screen

Created by Václav Brodec, last modified on Mai 10, 2016

Initial State

The not yet logged user arrives to the application landing page or is there sent by a redirect from one of the application pages unavailable out of session.

Description

  1. After arrival user sees login page (Mockup 1).

  2. User can input login credentials (email and password) and submit.

    1. As a result, application displays the content of  progress report mockup surrounded by a menu (with links to other top-level screens) and a footer and also a logout button.

    2. From now on, unless noted otherwise, only the inner content changes (the menu and other surrounding elements remain visible on all pages (as seen in Mockup 2).

Alternatives

  • In 2., a new user can click the Sign up button and follow to the sign up screen in Scenario 0b.

What can go wrong

  • The user of given login does not exist. (A text overlay suggesting the user to sign up appears.)
  • The input password does not match the login. (A text overlay suggesting the user to sign up appears.)
  • The connection either to service managing authentication or to service managing tasks is unavailable.

System state on completion

  • User is logged in.
  • User was given overview of the state of his tasks.

Requirements Motivating the Scenario

  •  7, 8, 44, 56
