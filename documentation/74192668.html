ADEQUATE : Story: User may lock disambiguation/classification for the given cell

Created by Václav Brodec, last modified by Tomas Knap on Sep 22, 2016

Github Issues: https://github.com/odalic/sti/issues/28 , https://github.com/odalic/odalic-ui/issues/95

User Story

As a user I want to be able to lock the selection of the resources classifying/disambiguation the given cell value. 

The choice is remembered for the re-runs.



Preconditions

  • The algorithm has been run at least once on the data or the application already knows which cells are available in the input.

Detailed Description

  1. User want to lock the given cell, so that next runs of the algorithm do now overwrite such selection.
    1. As a result, icon changes from unlocked to locked
    2. The choice is saved for the re-runs
  2. The user re-runs the algorithm.
  3. The icon for the cell must correspond to the actual state as it was before re-run (locked/unlocked)
  4. User may unlock the given cell 
    1. As a result, icon changes from locked to unlocked

Acceptance Criteria & Test Scenarios

  1. User may lock certain cell, icon is changed
  2. User may re-run alogirhtm 
  3. As a resul, icon remains correclty set up and odalic core does not try to re-classify/re-disambiguate
  4. User may un-lock,icon is properly changes

Variations

  • The user is able to take his decision back without any problems affecting the immediate export. No re-run is necessary - So by locking/unlocking, no auomtatic re-run is needed.
