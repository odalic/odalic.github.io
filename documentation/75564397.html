ADEQUATE : Story: User can propose relations

Created by Tomas Knap, last modified by Václav Brodec on Okt 05, 2016

Github Issues: https://github.com/odalic/sti/issues/188

Req: 74

User Story

As a user i want to be able to propose new relations in the primary knowledge base.

Currently, there is only possibility to define completely custom relation (at the level of UI).


Preconditions

  • The algorithm has been run at least once on the data or the application already knows which cells are available in the input.

 

Detailed Description 

 

  1. User can click on Propose icon in Step 3 (directly on the graph) 
    1. As a result, dialog appears where he can define further aspects of the proposed resource
  2. When satisfied, user clicks OK and the system tries to create newly proposed resource in the primary KB
    1. If successfull, see Step 3. If not, warning is shown and user has to adjust the definition of the new concept
  3. User is returned back to the graph - he may see his new resource being used for the annotation. 

 

    1. Also the relation is automaticall locked. See: Story: User may lock relations (at the level of all relations between two nodes)

 

 

 

Alternative:

 

  1. User can open details of relation in Step 3 (click on the existing relation)
    1. As a result, such dialog is open
  2. User can click on Propose button under the primary knowledge base section in the dialog
    1. As a result, dialog appears where he can define further aspects of the proposed resource
  3. When satisfied, user clicks OK and the system tries to create newly proposed concept in the primary KB
    1. If successfull, see Step 4. If not, warning is shown and user has to adjust the definition of the new resource
  4. User is returned back to the dialog - he may see his new resource being used for the annotation. New resource is selected. All other previous annotations are removed. 
      1. Also the relation is automaticall locked. See: Story: User may lock relations (at the level of all relations between two nodes)


Acceptance Criteria & Test Scenarios

  1. User may propose new relation
    1. Check that such relation is really created in primary knowledge base
  2. User can immediately see the result of proposing the relation - it is used for annotations
