ADEQUATE : Scenario 7b Last used in task opening

Created by Václav Brodec, last modified by István Satmári on Mai 22, 2016

Initial State

The user is logged to the application.

Description

  1. User enters the input file management screen.

  2. User clicks the open task button on the row with chosen file and the task creation dialog appears. (Mockup 1).

Alternatives

  • The task might not exist any more, so the "Last used in" cell can be empty.
  • The file may have been uploaded via a REST service and has not been used in a task yet. In this case, the "Last used in" cell is empty as well.

What can go wrong

  • The task might be removed before the page update.

System state on completion

Requirements Motivating the Scenario
