ADEQUATE : Story: Task configurations may be executed (on the server, results fetched later)

Created by Tomas Knap, last modified on Okt 12, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/106

Req: 

User Story

As a user I would like to be able to execute task configuration. 

This is available from the table of tasks  - where it is executed on the background and also from the task detail (where I can save and run the task, as also available in Iteration 2 - so I directly wait for the result), see: Story: Task configuration - User may edit task configuration

 

Preconditions

Detailed Description

  1. User may see execute the task. As a result, when done, user may examine result of the task - see: Story: Task configurations - last results may be explored


Acceptance Criteria & Test Scenarios

  • The processing continues even when the user ends the session, closes the browser.
  • User is not able to create multiple running or idle instances of one task configuration.
  • System reasonably limits the number of concurrently running instances. For the 3rd iteration only one running instance at a time (across the whole application) is acceptable.
  • Run action button changes to abort action immediately after it is clicked and behaves as such.
  • The available states are as specified in the REST API specification (REST API: 3rd iteration proposal).
  • Immediately after clicking the run button, the the status changes either into scheduled or currently running.
  • The states are periodically updated for the topmost task(s) in the execution order.
