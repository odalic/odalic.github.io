ADEQUATE : Story: Move subject column detection after classification/disambiguation step

Created by Tomas Knap, last modified by Václav Brodec on Okt 05, 2016

Github Issues: https://github.com/odalic/odalic-ui/issues/103

Req: 

User Story

As a user I would like to see results of classification and disambiguation first, then subject column detection.

So it is necessary to move subject column detection after classificaiton/disambiguation. 

We will see whether there is a need to have subject column detection table for each knowledge base, so that user can define different subject columns in different knowledge bases. For now, keep it as it is - one table with the possibility to select (common) subject column. 

Acception Criteria

  • The subject column selection step is the 2nd in order when applicable.
  • All alternative means of step changing (history, shortcuts,...) work as well.
