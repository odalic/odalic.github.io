ADEQUATE : Browsing/Searching Knowledge Bases

Created by Tomas Knap on Jul 15, 2016

What we are looking for: 

  • possibility to show in a tree/graph the resources in the RDF store (could be just resources with their atributes and typically rdf:type predicates)
    • with the possibility to distinguish classes from instances (based on the predicate rdf:type)
  • possibility to browse OWL/RDFS ontologies - classes, predicates, owl instances.

 

Summary

The best candidate for showing details of certain resource is LodView . See: Story: User can browse KB for a concept to be used for classification/disambiguation

The drawback of LodView is that it does not support search (you have to know URI in advanced). So It can be combined with  LODLive, which provide searching capabilities (keywords). See: Story: User can search KB for a concept to be used for classification/disambiguation/relation discovery

LodView can also be easily switched to LodLive (detail view to graph view). We can easily extend it so that also LodLive (graph view) can be switched back to the detail view (LodView).

Tools

LODLive

http://en.lodlive.it/

Source code: https://github.com/dvcama/LodLive
 

Navigator, accessing resources solely based on SPARQL standard. To browse RDF repositories.

LodLive is composed of a jQuery plug-in (lodlive-core.js), a JSON configuration map (lodlive-profile.js), an HTML page, a few images (sprites), and some other jQuery public plug-ins. It is as simple as that.

 

LodLive is an experimental project that was set-up to spread and promote the linked-open-data philosophy and to create a tool that can be used for connecting RDF browser capabilities with the effectiveness of data graph representation.

 

LodLive is the first navigator to use RDF resources based solely on SPARQL endpoints.

 

LodLive aims to demonstrate how resources published by W3C standards for the semantic web can be made easily accessible and legible with a few viable tools.

 

LodLive is capable of connecting the resources existing in its configured endpoints, allowing the user to pass from one endpoint to another by making use of LOD interconnection capacities.

They have keyword search.

When clicking start, the graph appears: 

You can see details of the resource, examine relations.. http://en.lodlive.it/?http://dbpedia.org/resource/Prague


Possible Improvements: Currently, you can just see number of relations coming out, but not type of such relations (only for sameAs and rdf:type). It would be good to show e.g. in tooltip the relations behind the small bullets. 

 

Summary: Provides keyword searching + browsing capabilities - graph based. 

LodView

http://lodview.it/

Example (Prague from dbpedia): http://lodview.it/lodview/?IRI=http%3A%2F%2Fdbpedia.org%2Fresource%2FPrague&sparql=http%3A%2F%2Fdbpedia.org%2Fsparql&prefix=http%3A%2F%2Fdbpedia.org%2F

Browsing Dbpedia ontology:

  • class: http://lodview.it/lodview/?sparql=http%3A%2F%2Fdbpedia.org%2Fsparql&prefix=http%3A%2F%2Fdbpedia.org%2F&IRI=http%3A%2F%2Fdbpedia.org%2Fontology%2FPlace
  • prerdicates: http://lodview.it/lodview/?sparql=http%3A%2F%2Fdbpedia.org%2Fsparql&prefix=http%3A%2F%2Fdbpedia.org%2F&IRI=http%3A%2F%2Fdbpedia.org%2Fontology%2FareaUrban

LodView is a Java web application based on Spring and Jena, it's a tool able to offer a W3C standard compliant IRI dereferenciation. LodView, in conjunction with a SPARQL endpoint, allows you to publish RDF data according to all defined standards for Linked Open Data.
LodView is easy to configure and deploy for any developer and it dramatically improves the enduser’s experience in accessing HTML based representations of RDF resources.

Code: https://github.com/dvcama/LodView

It is Java web application. Naslednik "pubby". Possibility to switch to LodLive browsing - see above. 

Drawback: It is mandatory to start with resource URI. Much nicer would be to support starting with full text search! 

Minor drawback: Instead of showing URIs of the predicates for certain resource, just title would be more userfriendly with URIs in the tooltips

Summary: Nice tool for browsing knowledge base (resources). Does not have any special support for browsing ontologies (classes, predicates). Active. Open License. No search, it has to start with URI.

Rhizomik

 

Rhizomer is a web application that facilitates publishing and also exploring Semantic Web data (and particularly Linked Data).

 

With Rhizomer, lay users can explore datasets performing the 3 basic data analysis tasks proposed by Shneiderman:

http://rhizomik.net/html/rhizomer/

http://rhizomik.net/html/rhizomer/install/

Summary: Very preliminary, not enough stable for us

 

jOWL

 

http://jowl.ontologyonline.org/

 

jOWL is a jQuery plugin for navigating and visualising OWL-RDFSdocuments. Javascritp library.

 

Summary: Not active, last issue in the tracker from 2012.

Graphity

http://atomgraph.com/?mode=http%3A%2F%2Fgraphity.org%2Fgc%23GridMode&limit=20

 

Graphity Browser is a fully extensible generic Linked Data platform for building end-user Web applications. It can be used for exploration and browsing of remote datasources, publishing and analysis of open data, as well as import and integration of private user data.

 

Building a data-intensive Web application on Graphity Browser is as simple as overriding generic stylesheets with own layout and defining necessary queries. The platform supports standard RDF access methods such as Linked Data and SPARQL endpoints, and includes plugin mechanisms for importing file formats and APIs as RDF.

Summary: Not usable, not an open source

 

GraphVizdb

http://83.212.97.26:8080/graphVizdb/

graphVizdb is a tool for efficient visualization and graph exploration operations over very large (RDF) graphs.

Summary: Too messy, no nice graph visualizations for normal users

rdf-ext

https://github.com/rdf-ext/rdf-ext

RDF-Ext provides a JavaScript library for working with RDF & Linked Data. This module contains the core classes to handle RDF Model data. Additional modules may be required to handle data in different formats (Turtle, JSON-LD, RDF/XML) or stores (LDP, SPARQL). 

rdfstore-js

rdfstore-js is a pure Javascript implementation of a RDF graph store with support for the SPARQL query and data manipulation language.
