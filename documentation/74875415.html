ADEQUATE : Story: Task configuration is saved upon creation

User Story

As a user when I create a new task, I expect it to be permanently available (to select from a list and run again) in the initial configuration until its modification.

TODO: Decide on the configuration storage.



Preconditions

  • All the parts of task configuration are set.

Detailed Description

  1. User confirms the task creation with the set configuration.
  2. A task is created.
  3. The task configuration is saved.
  4. Its processing continues as before.
  5. The task is available in the (newly established) list of available configured tasks to be run again.

Acceptance Criteria & Test Scenarios

  1. The configured task is available to the user even after server restarts and/or session renewal (if implemented).
  2. Restored task configuration is identical to the one saved.
  3. Unless the content of input files changes or the used knowledge bases are modified, the loaded task configuration gives the same results when run.
  4. Only a valid (adhering to specification) task configuration is stored.
