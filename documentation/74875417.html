ADEQUATE : Story: Task configuration - User may edit task configuration

Created by Václav Brodec, last modified by Tomas Knap on Sep 30, 2016

Github Issues: https://github.com/odalic/sti/issues/168 , https://github.com/odalic/odalic-ui/issues/98

Req: 32, 29

Related to: Story: Task configuration is saved upon creation

User Story

As a user when I change an existing task configuration, I expect it to be permanently available in the altered form until its next modification.



Preconditions

  • All the parts of task configuration are set.

Detailed Description

  1. User selects the configuration from the table
    1. he may either click on the row or click on the "detail icon"
  2. As a result, task configuraiton dialog appears
  3. User may change the configuraiton
  4. User may save or save and run the conf.
    1. If he clicks save and run, it behaves as now

Acceptance Criteria & Test Scenarios

  1. The altered configured task is available to the user even after server restarts and/or session renewal (if implemented).
  2. Restored task configuration is identical to the form it took after the latest modification.
  3. Unless the content of input files changes or the used knowledge bases are modified, the loaded task configuration gives results differing from the original only to the extent of altered options.
  4. Only a valid (adhering to specification) altered task configuration is stored.

Notes: 

When edited, last status may contain wrong results
