ADEQUATE : Scenario 2c Dataset Import - SPARQL

Created by Jan Váňa, last modified on Mai 23, 2016

Initial State

The user is logged on the system  (see: Scenario 0a Login and home screen)

Description

  1. User displays a list of knowledge bases (see Scenario 9)   
  2. User decides to add a new dataset to an existing knowledge base
    1. As a result, system shows the dialog from Mockup 1.
  3. User inserts the Dataset Name and Named Graph.
  4. User chooses to import from a SPARQL endpoint and confirms the dialog.
    1. As a result, system shows the dialog from Mockup 2.
  5. ser inserts the SPARQL endpoint and confirms the import
    1. As a result, system imports the dataset

What can go wrong

  • The specified SPARQL endpoint is not valid.

System state on completion

The system saves the dataset and the user can use it in further conversion tasks.

Requirements Motivating the Scenario

49.

Implementation Notes

SPARQL endpoint information is stored in the database.
