ADEQUATE : Scenario 4c Customizing disambiguation - selecting an extra concept for disambiguation

Created by Josef Janoušek, last modified by Tomas Knap on Sep 22, 2016

Initial State

The user is logged on the system (see: Scenario 0 Login and home screen)

The user wants to choose alternative disambiguation of respective cell (see: Scenario 4a Customizing subject column - Selecting different subject column)

Description

  1. User is provided with a table preview with the suggested winning disambiguation for the given cell value of certain CSV column

    1. System clearly marks the suggested winning choice (e.g. http://adequate.at/concept/city/Aibl for column with cities)
  2. User would like to specify that also the resource http://freebase.com/../Aibl is the right concept disambiguating the given cell value
    1. User selects "Customize" 
      1. As a result, system shows "Customize disambiguation" dialog as below

        1. List with alternative disambiguations suggested by the Odalic Core is shown, for each knowledge base in a different section

          1. options are sorted by probability / score

    2. User selects also the concept Aibl from Freebase KB

  3. User confirms the given selection

    1. As a result system saves such selection and changes the suggested winning concept for the disambiguation of the given cell value

Alternatives

  1. In 2. user may select a different (not suggested) concept by browsing/selecting the concept in knowledge base (in a similar way as in Scenario 4b)
  2. In 2. user may create new concept in the given internal knowledge base - as a result, system shows a dialog for specifying basic details about the concept
  3. In 2. when the user wants to provide and existing custom concept  - he may write appropriate custom URL to the "custom" textbox
  4. In 2. when the user isn't satisfied with any suggested alternative, he selects "no suitable" option in the customize disambiguation dialog, withou specifying the correct alternative - as a result, the table preview is updated and there is no disamibugation for the given cell value 

What can go wrong

There is no suggested disambiguation for the given cell value (due to low evidence in KBs). 

System state on completion

The user is provided with the alternative disambiguation of the cell value (the one he selects)

Requirements Motivating the Scenario

  • 18, 19, 20, 52

Notes

More than one concept may be used (one from each KB) to disambiguate the column

moreover, by clicking on the URL, user looks at the disambiguated entity in the context of the knowledge base (corresponding webpage is opened)
