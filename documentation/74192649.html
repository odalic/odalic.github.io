User Story

As a user I might want to keep selected cells from attempting disambiguation on them. The reasons may vary: either the available knowledge bases do not provide a suitable concept to map the value to, or the disambiguated concept negatively affects the classification of the whole column, or the text value itself is naturally ambiguous.

The choice is remembered for the re-runs, so unless I revert the choice back, all subsequent exports will keep the cell ambiguous.

The ambiguous cell may still appear in the result as an object in a triple consisting of a value from the subject column on the same row and a predicate determined by the class assigned to the column the ambiguous cell resides in.

When I decide leave a whole column ambiguous, individual disambiguation choices are ignored, but remembered in case I change my mind.

The option to leave a column ambiguous can be controlled either by a separate form element (checkbox), in that case making it checked disables all other associated controls for the cell.

Or the option is controlled by the same element that turns custom disambiguation on, in that case the element determining the custom disambiguation is set on a special value with the "leave ambiguous" meaning.



Preconditions

  • The algorithm has been run at least once on the data or the application already knows how many rows and columns are available in the input.

Detailed Description

  1. User indicates that a cell is to be left ambiguous (by unchecking associated check-box).
  2. When the user chooses to export the result, the cell appear in the result only in its ambiguous form (without assigned entity and only as a literal object).
  3. The choice is saved for the re-runs.
  4. The associated check-box is filled according to the choice when loaded in the next run.

Acceptance Criteria & Test Scenarios

  1. The choice to leave the cell ambiguous is applied in the subsequent export.
  2. Leaving a whole column ambiguous supersedes any individual disambiguation choices.
  3. Associated form elements are pre-filled according to the set choice when loaded.

Variations

  • Optional goal: The user is able to take his decision back without any problems affecting the immediate export. No re-run is necessary (unless other feedback, requiring a re-run is provided).
