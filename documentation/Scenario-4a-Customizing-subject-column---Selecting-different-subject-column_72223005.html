ADEQUATE : Scenario 4a Customizing subject column - Selecting different subject column

Created by Josef Janoušek, last modified by Tomas Knap on Sep 23, 2016

Initial State

The user is logged on the system (see: Scenario 0a Login and home screen)

User have defined and run a task (see: Scenario 1 Task creation). User is observing the results of the processing.

Description

As a user I want to change the subject column to another one, because the algorithm could make a completely wrong choice or there is a column, that I find more suitable to be the subject column.

As the changing of the subject column has a serious impact on the final result, I am OK with re-running the algorithm again in order to let the algorithm take the changed column into account.

 

  1. User is provided with the suggested subject column (one of the original CSV columns).

    1. Step 1 contains only subject column of the primary knowledge base (other KBs can have different subject columns. But this is OK, subject column influences the relations discovery - each KBs may discover different relations, and also export - in that )
  2. User may select different subject column as depicted in Mockup 1  (using the radio buttons).
    1. As a result, system records the selection and saves it as a part of the feedback of the task. (i.e. it does not change the resulting RDF file)
  3. If no other customization is done, user clicks on the "Re-execute task" button to execute task with provided feedback

Alternatives

  • Even when the suggested column is directly lifted from the (optionally) provided meta-data file, custom selection is still enabled.
  • User can check the "This is a statistical file..." option, thus making the subject column selection irrelevant and setting the feedback provision into RDF cube mode.
  • User can check the "Automatic selection" option (which is checked by default, until the user un-checks it or directly selects some other column), thus leaving the decision to the algorithm again.

What can go wrong

  • The wanted subject column is set to be ignored: the user must revert the decision to ignore the desired column first.

System state on completion

Different subject column is selected.
